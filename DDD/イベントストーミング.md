# イベントストーミング

実際の業務のワークフローを分析してドメインモデルを設計するための手法

ステークホルダー全員で集まって、付箋などを使って業務の流れを可視化する。
特に開発者とドメインエキスパートの両方がいる
## 進め方

大まかに分けて次の3つのステップがある。
上ほど参加者の範囲が広く、下になる程専門的になる。

### 1. ビックピクチャ: イベントを洗い出して時系列に並べる
1. Pivotal Event(後戻りできないイベント)をマークする
2. スイムレーン(並行処理, 条件分岐)を追加する
3. 関係者/外部システムを洗い出し
4. 全部終わったら、ナレーション/ウォークスルーを行う(認識合わせ)
### 2. ビジネスプロセスモデリング: コマンドやアクターを追加して業務フローを整理
### 3. ソフトウェアシステムモデリング: 制約や境界付けられたコンテキストの定義・深掘り

## 図の説明

付箋の色を変えて次の順番で貼っていく

1. ドメインイベント(オレンジ): ビジネス上の業務の出来事
2. ホットスポット(ピンク): 留意点・懸念点。いつでも追加可能
3. コマンド(青): イベントを引き起こす操作
4. アクター(黄色): コマンドを引き起こす主体
5. ポリシー(紫): ドメインイベントが発生した時の対応ルール
6. 集約(薄黄色): 一貫性のある単位?
